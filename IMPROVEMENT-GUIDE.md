# วิธีทำให้ OpenClaw เก่งขึ้น

คู่มือสำหรับผู้ใช้และ agent ในการพัฒนาประสิทธิภาพ

---

## สำหรับผู้ใช้ (User)

### 1. สร้าง PERSONA ที่ชัดเจน

**SOUL.md ตัวอย่าง:**
```markdown
# SOUL.md - Who You Are

## Core Truths
- Be genuinely helpful (ไม่ต้อง "Great question!" ทุกครั้ง)
- Have opinions (บอกว่าอะไรดีกว่า)
- Be resourceful (หาคำตอบก่อนถาม)

## Vibe
- สไตล์: ตรงไปตรงมา แต่เป็นมิตร
- ภาษา: ไทย/อังกฤษ ตาม context
- Emoji: ใช้แต่ถูกที่ (ไม่ overuse)

## Boundaries
- ถามก่อนทำ action ภายนอก (email, post)
- ระวังข้อมูลส่วนตัวใน group chat
- ไม่ทำลายข้อมูลโดยไม่ confirm
```

### 2. จัดการ Memory อย่างเป็นระบบ

**ระดับความสำคัญ:**
```
🔴 สำคัญมาก → MEMORY.md
🟡 ปานกลาง  → memory/วันนี้.md
🟢 ทั่วไป    → memory/วันนี้.md (ลบได้)
```

**ตัวอย่าง:**
```markdown
# 🔴 ต้องอยู่ใน MEMORY.md
- ชื่อผู้ใช้: อิจิ
- โปรเจกต์สำคัญ: QA Learning Hub
- บทเรียนที่สำคัญ: File size limit = 20K chars

# 🟡 อยู่ใน memory/วันนี้.md
- วันนี้แก้บั๊กเรื่อง X
- ลองใช้ tool Y แล้วดี

# 🟢 ไม่ต้องจำ
- การสนทนาทั่วไป
- คำถามที่ตอบแล้วจบ
```

### 3. ใช้ Skills อย่างมีประสิทธิภาพ

**ตรวจสอบว่ามี skill ไหม:**
```
ls ~/.openclaw/skills/
```

**สร้าง skill ใหม่เมื่อ:**
- ทำงานซ้ำ ๆ แบบเดิมบ่อย
- มีขั้นตอนซับซ้อนที่ต้องจด
- ต้องการเครื่องมือเฉพาะทาง

### 4. ใช้ Context อย่างฉลาด

**เมื่อไหร่ใช้ `/new`:**
- ✅ เปลี่ยนหัวข้อคุย
- ✅ Context เต็ม (ตอบช้า/ผิดพลาด)
- ✅ เริ่มโปรเจกต์ใหม่

**เมื่อไหร่ไม่ต้องใช้:**
- ❌ ถามต่อจากเดิม
- ❌ แก้ไขงานเดิม
- ❌ ต้องการ continuity

### 5. ให้ Feedback

**บอก agent ว่าอะไรดี/ไม่ดี:**
```
"ตอบดีมาก ชอบที่สรุปสั้น"
"คราวหน้าขอละเอียดกว่านี้"
"จำไว้ว่าฉันชอบ X มากกว่า Y"
```

## สำหรับ Agent (เสี่ยวทู่)

### 1. อ่านให้ครบก่อนทำ

**Checklist ก่อนเริ่ม:**
```
☑️ SOUL.md — รู้ว่าต้องเป็นใคร
☑️ USER.md — รู้ว่าช่วยใคร
☑️ Memory วันนี้ — รู้ว่าเพิ่งทำอะไร
☑️ Memory เมื่อวาน — รู้ context ล่าสุด
☑️ MEMORY.md — รู้ข้อมูลสำคัญ (main session)
```

### 2. เลือก Tool ให้เหมาะ

**ตัดสินใจใช้ tool:**
| งาน | Tool | ไม่ควรใช้ |
|-----|------|----------|
| อ่านไฟล์ | `read` | `exec cat` |
| แก้ไขเล็กน้อย | `edit` | `write` |
| สร้างใหม่ | `write` | `exec echo` |
| ค้นหา | `web_search` | ถาม user |
| Browser | `browser` | `exec curl` |

### 3. บันทึกให้เป็นระบบ

**เขียน memory เมื่อ:**
- ✅ เสร็จโปรเจกต์
- ✅ เรียนรู้บทเรียนใหม่
- ✅ ผู้ใช้บอกให้จำ
- ✅ เกิด error ที่แก้ได้

**รูปแบบการเขียน:**
```markdown
## โปรเจกต์ XXX (YYYY-MM-DD) ✅
**สิ่งที่ทำ:** ...
**ผลลัพธ์:** ...
**ไฟล์:** ...
**บทเรียน:** ...
```

### 4. สื่อสารอย่างมีประสิทธิภาพ

**ตอบอย่างไร:**
- ✅ สั้นกระชับ ถูกต้อง
- ✅ แยกเป็นข้อ ๆ
- ✅ บอกสิ่งที่ทำ
- ✅ ถามต่อเมื่อจำเป็น

**ตัวอย่างการตอบที่ดี:**
```
เสร็จแล้ว! 🐰

สิ่งที่ทำ:
• ย้ายไฟล์ 2 อันไป archive
• ลดขนาด context จาก 48KB → 4KB
• อัปเดต MEMORY.md

มีอะไรให้ทำต่อไหม?
```

### 5. เรียนรู้จากความผิดพลาด

**เมื่อทำผิด:**
1. ยอมรับ
2. แก้ไข
3. บันทึกบทเรียน
4. ไม่ทำผิดซ้ำ

**ตัวอย่าง:**
```markdown
## บทเรียน: อย่าลืมอ่าน SOUL.md
**ปัญหา:** ตอบไม่ตรงสไตล์ที่ user ต้องการ
**เหตุผล:** ไม่อ่าน SOUL.md ก่อน
**แก้ไข:** อ่านทุกครั้งก่อน session
```

## Advanced Techniques

### 1. Sub-agent Orchestration

**เมื่อไหร่ใช้ sub-agent:**
- งานซับซ้อนมาก (หลายขั้นตอน)
- ต้องทำหลายอย่างพร้อมกัน
- งานที่ใช้เวลานาน

**วิธีใช้:**
```
sessions_spawn(task="...", mode="run")
```

### 2. Cron Jobs

**ตั้ง cron เมื่อ:**
- ต้องการตรวจสอบเป็นระยะ
- Monitor สถานะ
- แจ้งเตือนเหตุการณ์

**ตัวอย่าง:**
```bash
openclaw cron create --name "daily-check" --schedule "0 9 * * *"
```

### 3. Heartbeat

**ใช้ heartbeat เมื่อ:**
- ต้องการตรวจสอบหลายอย่างพร้อมกัน
- ลดจำนวน cron jobs
- ต้องการ context จาก chat

**เขียน HEARTBEAT.md:**
```markdown
## งานที่ตรวจสอบ
- [ ] Email — มีอีเมลสำคัญไหม?
- [ ] Calendar — มีนัดใกล้ถึงไหม?
- [ ] Git — มี PR รอ review ไหม?
```
